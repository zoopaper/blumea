Introduction

Welcome to Melon, a responsive, flat and full featured admin template build exclusively for sale at ThemeForest. This template is build on Bootstrap 3 framework using modern techniques like HTML5 and CSS3 to be used for backend solutions of any size.

Support Requests

Browser support is targeted at IE8+, Google Chrome, Mozilla Firefox, Apple Safari, Opera and all Android and iOS mobile browsers. For browsers not in this list, support is not available.

Support requests are handled over my support forums at http://support.stammtec.de.

Documentation Scope

This documentation is targeted to explain structures like file and directories, template, CSS and JavaScript or how you can configure the template. Since this template is build on top of the excellent Bootstrap 3 framework, you main source of documentation is here: http://getbootstrap.com/css/

This documentation is also written in a style that assumes you already know the basics of CSS, JavaScript and HTML, so only the usage and concept of this template's elements that's going to be covered here.

Getting Started

To get started, just browse through the demonstration pages. Use Chrome Developer Tools (Chrome), Firebug (Firefox) or similar plugins to analyze the code. To get a deeper understanding of the code, open the HTML and JS files and read through the commented code.

Use the empty.html and start right away building your own pages.

Throughout the documentation, #ID refers to an element with id="ID"; and .class to an elements with class="..." containing .class. In addition, values in [square brackets] are values to be replaced by your code.

Template

All template files have a fixed structure consisting of header, sidebar and content.

Beginning of Page

Below code is used at the beginning of all HTML5 pages. You should change the lang="en" to your language.

<!DOCTYPE html>
<html lang="en">

handleProjectSwitcher() Adds functionality for project switcher at the header

Global Functions (e.g. for use in other files)

Widgets

Unboxed

Widgets in Melon are very easy to use and are flexible with a lot of options. General markup (unboxed example):

<div class="widget">
    <div class="widget-header">
        <h4><i class="icon-reorder"></i> Lorem Ipsum</h4>
    </div>
    <div class="widget-content">
        <p>Lorem ipsum dolor sit amet, consetetur sadipscing elitr.</p>
    </div>
</div>

Boxed

Now you can make the widgets appear in a box by simply adding the class .box to <div class="widget">, like in this example (see line 1):

<div class="widget box">
    <div class="widget-header">
        <h4><i class="icon-reorder"></i> Lorem Ipsum</h4>
    </div>
    <div class="widget-content">
        <p>Lorem ipsum dolor sit amet, consetetur sadipscing elitr.</p>
    </div>
</div>

If you want to remove the padding from boxed-widgets (useful for tables), you can add the class .no-padding to <div class="widget-content">:

[...]
    <div class="widget-content no-padding">
        <p>Lorem ipsum dolor sit amet, consetetur sadipscing elitr.</p>
    </div>
[...]

Toolbar

If you want to add a toolbar with buttons and/ or dropdowns, you can do it like that:

<div class="widget box">

        <!--=== Toolbar ===-->

                <span class="btn btn-xs widget-refresh"><i class="icon-refresh"></i></span>
                <span class="btn btn-xs"><i class="icon-plus"></i> Add</span>
                <span class="btn btn-xs dropdown-toggle" data-toggle="dropdown">
                    Manage <i class="icon-angle-down"></i>
                </span>
                <ul class="dropdown-menu pull-right">
                    <li><a href="#"><i class="icon-pencil"></i> Edit</a></li>
                    <li><a href="#"><i class="icon-trash"></i> Delete</a></li>
                    <li><a href="#"><i class="icon-ban-circle"></i> Ban</a></li>
                </ul>
            </div>

Toolbars works in unboxed and boxed widgets.

Navigation

Navigation works only in the sidebar and has the following markup:

<ul id="nav">
    <li>
        <a href="#">
            <i class="icon-angle-right"></i>
            Entry 1
        </a>
    </li>
    <li class="current open">
        <a href="#">
            <i class="icon-angle-right"></i>
            Entry 2
            <span class="label label-info pull-right">6</span>
        </a>
        <ul class="sub-menu">
            <li>
                <a href="#">
                <i class="icon-angle-right"></i>
                Entry 2.1
                </a>
            </li>
        </ul>

The entry with the right blue (default) border has the class .current. If a main entry has a submenu and the submenu should be visible on page-load, just add .open-default to the corresponding <li>.

Dynamic/ Managed Tables

Melon supports the super cool DataTables plugin for dynamic/ managed tables. To turn the static table into a dynamic table, you can simply add the class .datatable to the <table>-element. Now you have the default options. If you want to have more functions, you can add these classes:

Class Description
.table-tabletools TableTools adds a toolbar to a table to provide options for saving files (CSV, XLS, PDF), copying to system Clipboard, or printing the table. Row selection options are also available.
.table-colvis ColVis presents the end user with a list of columns in the table and options to enable or disable their current visibility.
.table-checkable Sorting will be disabled at the first column
.table-responsive Add responsive functionality to DataTables

Additionally, you can set some options directly in your HTML:

Attribute Description
data-display-length="10" Default number of entries displayed at the DataTable
data-horizontal-width="150%" Wider DataTable with horizontal scrolling

You can use these attributes like:

<table class="table datatable" data-horizontal-width="150%">

Other DataTable-Configurations like Ajax-Source

Since v1.1.0 you can simply add any paramteres that DataTables knows in data-dataTable='{}', for example for sAjaxSource:

<table class="table datatable" data-dataTable='{"bProcessing": true, "bServerSide": true, "sAjaxSource": "../examples_support/server_processing.php"}'>

Responsive Tables

  1. Add class .table-responsive to the <table>
  2. Add the data-class="expand" attribute to the th element for the respective column that will you want to display the expand icon in. This th elment cannot be a column that will be hidden.
  3. Add data-hide="phone,tablet" to the th element for the respective column that will you want to hide when the window is resized.
<table class="table table-responsive datatable">
    <thead>
        <tr>
            <th data-class="expand">First Name</th>
            <th>Last Name</th>
            <th data-hide="phone">Username</th>
            <th data-hide="phone,tablet">Status</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Joey</td>
            <td>Greyson</td>
            <td>joey123</td>
            <td>Approved</td>
        </tr>
    </tbody>
</table>

Forms

Basically you can decide between horizontal and vertical forms.

Horizontal Forms

General markup:

<form class="form-horizontal" action="#">

  • .input-width-xlarge
  • .input-width-xxlarge

        <label class="control-label">Lorem Ipsum:</label>

Recompile LESS

First of all you should be aware of the structure from the LESS-file, which you will see in the section "Files and Directories".

So you will see that whenever you change a file in _less/general/ you must only recompile _less/main.less because it imports all other files. Same with _less/general/variables.less. Save the compiled less as assets/css/main.css.

However for plugins there is another behaviour, because each file in _less/plugins/ is manually compiled and saved into assets/css/plugins/. If you add a new file for a plugin, remember to include it into the assets/plugins.css which is not compiled via less.

Remember: If you make changes directly in the CSS and/ or LESS-files from Melon, it will be hard to update if I release an update. So I would recommend to create a new CSS/ LESS-file for this purpose.

less-monitor (*nix only)

https://github.com/gdupont/less-monitor

If you want to automate your less-compiling, you can install a watchdog, which will automatically compile your file when it's saved. For this purpose I can definitely recommend less-monitor.

Monitor and recompile your .less files and dependencies.

If you're using less-monitor, you can use this commands:

less-monitor -d "/your_path/_less/" -o "../assets/css/" --master true --ignore "**/plugins"
less-monitor -d "/your_path/_less/plugins/" -o "../../assets/css/plugins/" --master true

Note: I'm not giving support for this application neither for the installation or configuration. I just recommend this little tool for everyone. If you need help, look at their GitHub page.

Change Color

Dark Theme

I provide you with a dark theme, where mainly the sidebar is dark blue. To activate it, add the class .theme-dark to the <body>:

<body class="theme-dark">

Own Colors

You can completely change the default template color (blue) to any color you want.

Open _less/general/variables.less and change @mainColor. After that, you should recompile your main.less and all .less's in _less/plugins/. See section Recompile LESS.

Own CSS

As already mentioned in Recompile LESS, it would be a better idea to outsource all your additional CSS to another file, because it will be hard to update if I release an update. It would overwrite your changes.

Simply create a new file e.g. called assets/css/custom.css and include it in your HTML-files. Make sure to include it after all the other CSS-files (e.g. main.css to overwrite CSS-rules).

<link href="assets/css/custom.css" rel="stylesheet" type="text/css" />

Miscellaneous

Disable resizeable sidebar

Remove the class .resizeable from <div id="divider">. Example:

<div id="divider"></div>

Row Background

You can simply put any content in a row with a fancy background. You need to add the class .row-bg to the <div class="row">. Example:

<div class="row row-bg">

Hide / Remove Sidebar

To remove the sidebar from specific pages, you can remove the #sidebar-div-container (this is optional!) and then add the class .sidebar-closed to the #container-div, e.g.:

<div id="container" class="fixed-header sidebar-closed">

Frequently Asked Questions (FAQ)

I get the JavaScript-error Uncaught TypeError: Cannot set property 'w' of undefined.

This is an error in the flot-resize library (jquery.flot.resize.js). To solve it, simply move the flot-libraries before all the other jQuery-plugins (but after jQuery).

Why do you put all the JavaScript at the top? All cool guidelines are saying: "Put it at the bottom".

Well, this guidelines are for landing pages, company pages and so on. If you put the JavaScript at the bottom, the browser will start rendering the text and afterwards it will load the JavaScript. I agree, this is perfect for nearly all sites, excluding backend solutions or admin templates.

But nearly all admin templates need the JavaScript otherwise the user can perform some actions where actually JavaScript should step in which will result in a bad user experience.

Note: The FAQ will grow in time as buyers ask questions.
